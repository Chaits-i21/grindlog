Grindlog Privacy Policy

Last updated: July 2026

The short version: Grindlog collects nothing. It has no servers, no analytics, no accounts, and no tracking of any kind. Everything happens in your browser.

What Grindlog stores, and where

  • Your GitHub token and repository settings — stored in Chrome's extension storage (chrome.storage.sync) on your device / Chrome profile. Never sent anywhere except GitHub's API.
  • Per-problem solve tracking (time opened, failed-attempt counts) and the retry queue — stored locally in chrome.storage.local on your device.

What Grindlog sends over the network

  • To leetcode.com — requests for problem metadata (statement, difficulty, topics, hints) and, if you use the import feature, your own submission history. These use your existing LeetCode login session; Grindlog never sees your LeetCode password.
  • To api.github.com — your solution files and generated READMEs, committed to the repository you configured, authenticated with the token you created.

That is the complete list. Grindlog makes no other network requests.

What Grindlog does not do

  • No data is sent to the developer or any third party.
  • No analytics, telemetry, cookies, or fingerprinting.
  • No reading of browsing activity outside leetcode.com problem pages.
  • Your code is never stored anywhere except your own GitHub repository.

Your control

Removing the extension deletes all locally stored data. You can revoke the GitHub token at any time from your GitHub settings, which immediately cuts off Grindlog's ability to push.

Contact

Questions or concerns? Open an issue on the GitHub repository.
